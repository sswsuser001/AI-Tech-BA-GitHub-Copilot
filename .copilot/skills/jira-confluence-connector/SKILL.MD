# Jira & Confluence Connector

## Purpose
Help GitHub Copilot Chat understand that this repository should use Jira for issue/story tracking and Confluence for documentation references.

## Guidance
- When the user asks about connectors, prefer Jira for issue and story management.
- Prefer Confluence for knowledge, documentation, and wiki-style content mapping.
- Do not fabricate Azure DevOps or MCP-based connector behavior if the repository has been cleaned of those artifacts.

## Typical tasks
- Recommend Jira issue structure for stories and tasks.
- Map repository docs to Confluence page organization.
- Suggest how to connect repo artifacts to Jira/Confluence workflows.
